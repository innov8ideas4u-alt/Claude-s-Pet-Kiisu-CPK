## 1. Goals & non-goals

### 1.1 Goals

**G1.** Host can receive unsolicited frames from the FAP (command_id == 0) and route them to per-opcode handlers without breaking existing request-response flows.

**G2.** FAP can do heavy hardware work (NFC reader, future SubGHz/IR) on a worker thread without blocking the RPC callback context or other RPC traffic.

**G3.** End-to-end validation: a host-side subscription receives NFC card-tap events as unsolicited broadcasts emitted by the FAP's worker thread.

**G4.** All Phase 2.5 tools continue to pass their 27 tests after the host-side refactor. Zero regression.

### 1.2 Non-goals

- **Not** building SubGHz, IR, or GPIO opcodes in Phase 3 (deferred to Phase 4)
- **Not** building multi-subscriber refcounting (deferred to Phase 4 if needed)
- **Not** building host-side keepalive or USB-disconnect detection (per Q6 — firmware flow control suffices)
- **Not** building mJS async-iterator API (mJS doesn't support async/await; shipping blocking wait_event instead per Q5)
- **Not** patching Momentum's uninit-malloc bug in this phase (separate PR, deferred)

## 2. Architecture summary

**Before (Phase 2.5):** One host operation at a time. Host calls a public RPC method, the wire-lock decorator holds the lock across the whole await, the method does send-then-receive serially, returns. The wire is owned by whoever holds the lock.

**After (Phase 3):** Single reader task owns ALL reads from the wire. Public methods acquire the wire-lock only for the brief send() call, then await on a per-command_id future. The reader task demultiplexes incoming frames by command_id: positive IDs match pending futures, zero IDs route to the subscription layer.

On the FAP side: a FuriThread worker does hardware (NFC reader). Worker queues results to a FuriMessageQueue consumed by the main thread. ONLY the main thread calls rpc_system_app_exchange_data() — worker never touches the wire.

The NFC slice is the proof-of-life: host subscribes, FAP worker arms reader, taps generate broadcasts, host listener dispatches to mission callback. F4 (broken require("nfc") in mJS) is retired.

## 4. Host-side refactor: single-reader pattern (THE BIG ONE)

### 4.1 Current state (Phase 2.5)

In `flipper_mcp/core/protobuf_rpc.py` lines 38-82, the `@_with_wire_lock` decorator currently wraps the entire public method body:

```python
async with self._wire_lock:
    return await method(self, *args, **kwargs)  # send + recv both inside lock
```

This means: every Phase 2.5 RPC method holds the wire mutex from "before send" to "after final response byte is received." That works perfectly when only one consumer ever reads the wire. It is fundamentally incompatible with a background listener task that needs to read the wire continuously.

### 4.2 New state (Phase 3)

The wire-lock changes meaning. It becomes a **send-only mutex** that serializes the brief moment of writing bytes to the wire. After send returns, the lock is released, and the public method awaits on a per-command_id future. A single background reader task is the sole reader of the wire; it demuxes all incoming frames.

```python
async def _public_rpc_method(self, ...):
    cmd_id = self._next_command_id()
    future = asyncio.Future()
    self._pending[cmd_id] = future
    try:
        async with self._wire_lock:               # held only for send
            await self.transport.send(framed_request_bytes)
        # lock released. wait for the reader task to populate the future.
        return await asyncio.wait_for(future, timeout=RPC_TIMEOUT_S)
    finally:
        self._pending.pop(cmd_id, None)
```

### 4.3 The reader task

A single asyncio.Task started at connect() time. It is the only thing that calls self.transport.receive().

```python
async def _reader_loop(self):
    while not self._stop_event.is_set():
        try:
            frame = await self.transport.receive(timeout=READER_POLL_S)
            if frame is None:
                continue
            main = parse_main(frame)
            cmd_id = main.command_id
            tag = main.WhichOneof('content')

            if cmd_id > 0:
                future = self._pending.get(cmd_id)
                if future and not future.done():
                    future.set_result(main)
                else:
                    logger.debug("stale frame cmd_id=%d tag=%s", cmd_id, tag)
                    continue

            elif cmd_id == 0:
                self._dispatch_broadcast(main, tag)

        except asyncio.CancelledError:
            break
        except CfcProtocolDesyncError:
            self._stop_event.set()
            raise
        except Exception:
            logger.exception("reader loop error")
```

### 4.4 Broadcast dispatch

```python
def _dispatch_broadcast(self, main, tag):
    if tag != 'app_data_exchange_request':
        # Per Phase 2.5 allowlist: 'empty', 'app_state_response',
        # 'gui_screen_frame', 'desktop_status' are legal cmd_id=0 frames.
        return

    payload = main.app_data_exchange_request.data
    cfc_header, msgpack_payload = parse_cfc_frame(payload)
    op_code = cfc_header.op_code

    subscription = self._subscriptions.get(op_code)
    if subscription is None:
        logger.debug("event op_code=%d with no subscriber", op_code)
        return

    try:
        subscription.queue.put_nowait((cfc_header, msgpack_payload))
    except asyncio.QueueFull:
        subscription.overflow_count += 1
        try:
            subscription.queue.get_nowait()
        except asyncio.QueueEmpty:
            pass
        subscription.queue.put_nowait((cfc_header, msgpack_payload))
```


### 4.5 New public API: flipper_cfc_listen MCP tool

```python
async def flipper_cfc_listen(op_code: int, timeout_ms: int = 5000) -> dict | None:
    """
    Wait for the next event matching op_code. Returns the event payload as a
    dict, or None if no event arrives within timeout_ms.
    Caller must first call flipper_cfc_subscribe(op_code).
    """
    sub = rpc._subscriptions.get(op_code)
    if sub is None:
        raise CfcNotSubscribedError(op_code)

    try:
        cfc_header, payload = await asyncio.wait_for(
            sub.queue.get(), timeout=timeout_ms / 1000.0
        )
        return {
            "op_code": op_code,
            "payload": msgpack.unpackb(payload),
            "overflow_count_so_far": sub.overflow_count,
        }
    except asyncio.TimeoutError:
        return None
```

Plus `flipper_cfc_subscribe(op_code)` and `flipper_cfc_unsubscribe(op_code)` MCP tools.

### 4.6 Migration of Phase 2.5 tools

Every public method currently decorated with `@_with_wire_lock` gets migrated to the new pattern. The decorator is rewritten to allocate cmd_id and a future, then call the underlying method, which acquires the lock only for send and awaits on the future for the response.

## 5. FAP-side worker pattern

### 5.1 Structure

In cfc/cfc.c, add:

```c
typedef enum {
    CfcWorkerEventTypeStop,
    CfcWorkerEventTypeNfcArm,
    CfcWorkerEventTypeNfcDisarm,
} CfcWorkerEventType;

typedef struct {
    CfcWorkerEventType type;
} CfcWorkerEvent;

typedef struct {
    FuriThread* thread;
    FuriMessageQueue* worker_in;   // main -> worker
    FuriMessageQueue* worker_out;  // worker -> main
    bool nfc_armed;
    uint32_t last_event_ms;        // for 5-min idle-stop
} CfcWorker;
```

### 5.2 Worker thread function

```c
static int32_t cfc_worker_thread(void* context) {
    CfcWorker* w = context;
    while(true) {
        CfcWorkerEvent ev;
        FuriStatus s = furi_message_queue_get(w->worker_in, &ev, 100);
        if(s == FuriStatusOk) {
            if(ev.type == CfcWorkerEventTypeStop) break;
            if(ev.type == CfcWorkerEventTypeNfcArm) w->nfc_armed = true;
            if(ev.type == CfcWorkerEventTypeNfcDisarm) w->nfc_armed = false;
        }
        if(w->nfc_armed) {
            NfcCard card;
            if(nfc_poll_once(&card, 50)) {
                CfcWorkerResult result = { /* card UID, timestamp */ };
                furi_message_queue_put(w->worker_out, &result, 0);
            }
        }
    }
    return 0;
}
```

### 5.3 Main thread drains worker_out

```c
// Called from main thread event loop, NOT from worker
static void cfc_drain_worker_results(CfcContext* ctx) {
    CfcWorkerResult result;
    while(furi_message_queue_get(ctx->worker.worker_out, &result, 0) == FuriStatusOk) {
        uint8_t frame_buf[CFC_MAX_FRAME_SIZE];
        size_t frame_len;
        cfc_build_event_frame(CFC_OPCODE_NFC_EVENT, &result, frame_buf, &frame_len);
        // command_id=0 set internally by rpc_system_app_exchange_data
        rpc_system_app_exchange_data(ctx->rpc_app, frame_buf, frame_len);
    }
}
```

### 5.4 Stack size

Per NotebookLM Round 5: **2048 bytes** for an NFC-class worker.

### 5.5 5-minute idle-stop failsafe (Q6)

Main thread tracks `last_event_ms` (timestamp of last host activity affecting worker). If `furi_get_tick() - last_event_ms > 300_000` AND worker is armed, push stop to worker_in and tear it down.

### 5.6 Shutdown sequence

```c
// 1. Disarm hardware first
// 2. Signal stop
// 3. furi_thread_join
// 4. furi_thread_free + furi_message_queue_free (both queues)
```

## 6. NFC vertical slice (NFC_SUBSCRIBE_CAPTURE)

### 6.1 Opcodes (new)

```
CFC_OPCODE_NFC_SUBSCRIBE_CAPTURE  = 0x40
CFC_OPCODE_NFC_UNSUBSCRIBE         = 0x41
CFC_OPCODE_NFC_EVENT               = 0x42  // event broadcast
```

### 6.2 Error codes (new)

```
CFC_ERR_BUSY              = 0x10  // Q2: exclusive subscription
CFC_ERR_NOT_SUBSCRIBED    = 0x11  // unsubscribe with no active sub
CFC_ERR_WORKER_BUSY       = 0x12  // worker thread stop in flight
```

### 6.3 Subscribe handler (FAP side)

```c
// Handle CFC_OPCODE_NFC_SUBSCRIBE_CAPTURE
if(ctx->worker.nfc_armed) {
    cfc_send_error_response(ctx, cmd_id, CFC_ERR_BUSY);
    return;
}
// Send ack FIRST (Q1: subscription becomes active after ack)
cfc_send_ok_response(ctx, cmd_id);
// THEN arm the worker
CfcWorkerEvent ev = { .type = CfcWorkerEventTypeNfcArm };
furi_message_queue_put(ctx->worker.worker_in, &ev, 100);
ctx->worker.last_event_ms = furi_get_tick();
```

### 6.4 Event broadcast payload format (msgpack)

```python
{
    "uid": bytes,              # 4 or 7 byte NFC UID
    "type": str,               # "iso14443a-4" / "mifare-classic" / etc.
    "rssi": int | None,
    "timestamp_ms": int,       # FAP tick at detection
    "overflow_since_last": int # events dropped since last delivered
}
```

### 6.5 mJS API (Q5)

```javascript
let cfc = require("cfc");
let sub = cfc.nfc_subscribe();
if (!sub) { log("subscribe failed"); return; }

while (true) {
    let ev = sub.wait_event(5000);
    if (!ev) { log("no event in 5s"); break; }
    log("tag uid=" + ev.uid.toString() + " type=" + ev.type);
    if (ev.overflow_since_last > 0) {
        log("WARN dropped " + ev.overflow_since_last.toString());
    }
}
sub.unsubscribe();
```

## 8. Wire protocol additions

### 8.1 CFC frame header (unchanged from Phase 1)

16 bytes: magic(2) + version(1) + op_code(1) + transaction_id(4) + fragment_index(2) + fragment_total(2) + payload_length(4)

### 8.2 New opcode allocations

- `0x40` NFC_SUBSCRIBE_CAPTURE (request)
- `0x41` NFC_UNSUBSCRIBE (request)
- `0x42` NFC_EVENT (broadcast, command_id=0)

### 8.3 New error codes

- `0x10` CFC_ERR_BUSY
- `0x11` CFC_ERR_NOT_SUBSCRIBED
- `0x12` CFC_ERR_WORKER_BUSY

### 8.4 Broadcast frame conventions

- `command_id` MUST be 0
- `transaction_id` in the CFC header MUST be incrementing
- `fragment_index`/`fragment_total` work the same as for responses

## 13. v2 PATCHES (folded in from Sherpa adversarial review 2026-05-27)

### 13.1 [CRITICAL] Reader loop must reassemble multi-fragment responses

The §4.3 reader loop as drafted calls `future.set_result(main)` immediately on every frame. CFC responses can span multiple fragments (transaction_id consistent, fragment_index 0..N-1, fragment_total=N). This breaks every existing CFC tool. Corrected reader logic:

```python
async def _reader_loop(self):
    # Per-transaction-id assembly buffers for in-flight multi-frag responses/events
    self._assembling = {}  # cmd_id -> dict(fragments=[], fragment_total=N)

    while not self._stop_event.is_set():
        try:
            frame = await self.transport.receive(timeout=READER_POLL_S)
            if frame is None:
                continue
            main = parse_main(frame)
            cmd_id = main.command_id
            tag = main.WhichOneof('content')

            if cmd_id > 0:
                # Response — may be multi-fragment
                self._accumulate_response(cmd_id, main)
            elif cmd_id == 0:
                # Broadcast — may be multi-fragment (per §8.4)
                self._accumulate_broadcast(main, tag)

        except asyncio.CancelledError:
            break
        except CfcProtocolDesyncError:
            self._stop_event.set()
            raise
        except Exception:
            logger.exception("reader loop error")


def _accumulate_response(self, cmd_id, main):
    # Non-CFC frames (e.g. plain Main responses from Phase 2.5 tools) are
    # single-frame; set the future immediately if no CFC fragmentation applies.
    tag = main.WhichOneof('content')
    if tag != 'app_data_exchange_request':
        future = self._pending.get(cmd_id)
        if future and not future.done():
            future.set_result(main)
        return

    # CFC response — check CFC header for fragmentation
    payload = main.app_data_exchange_request.data
    cfc_header = parse_cfc_header(payload)
    body = payload[CFC_HEADER_LEN:]

    if cfc_header.fragment_total == 1:
        # Single-fragment, deliver immediately
        future = self._pending.get(cmd_id)
        if future and not future.done():
            future.set_result((cfc_header, body))
        return

    # Multi-fragment — accumulate
    asm = self._assembling.setdefault(cmd_id, {
        'fragments': [None] * cfc_header.fragment_total,
        'fragment_total': cfc_header.fragment_total,
        'header': cfc_header,
    })
    asm['fragments'][cfc_header.fragment_index] = body
    if all(f is not None for f in asm['fragments']):
        full_payload = b''.join(asm['fragments'])
        future = self._pending.get(cmd_id)
        if future and not future.done():
            future.set_result((asm['header'], full_payload))
        del self._assembling[cmd_id]
```

`_accumulate_broadcast` follows the same pattern but uses transaction_id (not cmd_id) as the assembly key and dispatches to subscription queue when complete.

**This is a host-side change only.** Replaces §4.3 and §4.4 entirely.

### 13.2 [HIGH] Q1 ack-timing race fix

§6.3 has the FAP send ack, then arm worker. There's a window where the worker arms and detects a card before the ack reaches the host. Fix:

```c
// Handle CFC_OPCODE_NFC_SUBSCRIBE_CAPTURE
if(ctx->worker.nfc_armed) {
    cfc_send_error_response(ctx, cmd_id, CFC_ERR_BUSY);
    return;
}
// 1. Send ack to host
cfc_send_ok_response(ctx, cmd_id);
// 2. Yield briefly so ack is on the wire before we arm
furi_delay_ms(20);
// 3. NOW arm the worker
CfcWorkerEvent ev = { .type = CfcWorkerEventTypeNfcArm };
furi_message_queue_put(ctx->worker.worker_in, &ev, 100);
ctx->worker.last_event_ms = furi_get_tick();
```

Belt-and-suspenders: host-side `flipper_cfc_subscribe` records a timestamp `subscribed_at_ms`. Any event with `timestamp_ms < subscribed_at_ms` is dropped at the dispatcher. Implements Q1 strictly even if the FAP delay is insufficient.

## 17. v5 PATCHES (adversarial review of Cook 1.5 bugs — Arena dual-model, 2026-05-27)

Two independent models reviewed the two Cook 1.5 live bugs and converged on the following. These are MANDATORY for Cook 2 onward.

### 17.1 [CRITICAL] Reader must never block on a full subscription queue
The reader task is the sole wire consumer. If it ever `await queue.put()` on a full queue (slow/bursty consumer — ours is AI inference), the reader suspends, USB stops being read, and all pending requests hang. Host appears dead. Delivery into subscription buffers MUST be non-blocking.

### 17.2 [CRITICAL] Subscription buffer = deque(maxlen=N), atomic drop-oldest
Do NOT use asyncio.Queue with check-then-act drop ("if full: get_nowait(); put_nowait()") — a consumer can steal the slot between get and put → lost events/desync. Use collections.deque(maxlen=SUBSCRIPTION_QUEUE_DEPTH) (drops oldest atomically) + asyncio.Event to wake the consumer. _listen awaits the Event, drains the deque. overflow_since_last increments per drop, surfaced in event payload.

### 17.3 [CRITICAL] Transaction ID namespace partitioning (32-bit txn)
- Host-initiated request txns: 0x00000000–0x7FFFFFFF (high bit CLEAR)
- Firmware-initiated broadcast txns: 0x80000000–0xFFFFFFFF (high bit SET)
FAP allocates broadcast txns high-bit-set; host allocates request txns high-bit-clear. Dispatcher ASSERTS it: a broadcast (cmd_id==0) with high-bit-clear txn, or a response with high-bit-set txn, is dropped + logged as a protocol bug. Makes broadcast-vs-request Future collision structurally impossible.

### 17.4 [HIGH] Stale-subscriber safety
A subscriber may unsubscribe between the dispatcher's lookup and delivery. Snapshot the reference, guard delivery, never crash the reader on a missing/closed subscription — log and continue.

### 17.5 [HIGH] Chaos mocks (defense for the Cook 1.5 over-fitted-mock bug class)
Tests must not use orderly fake frames. Synthesize adversarial frames: garbage/high-bit-wrong txns, out-of-order fragments, duplicate fragments, zero-length payloads, split reads. Rule: if the firmware CAN send it (even as a bug), the test must eventually send it.

### 17.6 Cook 3 firmware concurrency (forecast — design against now)
Full detail in D:\Dev\scratch\day11_arena_bug_reflection.md. Summary:
- Wire interleaving: worker + main both writing USB = corruption. Worker NEVER writes; only enqueues to worker_out; main thread is sole sender (already our Q3 decision — reaffirmed).
- Use-after-free: worker must heap/pool-allocate the event payload and hand ownership to the sender, never pass stack pointers across threads.
- Atomic txn counter: use __atomic_fetch_add; worker allocates from the 0x80000000+ range only.
- ISR context: NFC tap handler (if in interrupt context) cannot malloc/pb_encode/USB — defer to worker via lock-free queue.
- USB backpressure: worker enqueues non-blocking (FuriWaitNone), drops if full (drop-newest OK for NFC), never blocks on USB.

## ⚠️ COOK 3 MUST-DO #1 — Wire interleaving (deferred from Cook 2, now REAL)

Cook 2 deferred the single-writer wire mutex because in the MOCK flow, responses (callback
thread) and broadcasts (main thread) were temporally separated — a 2s timer can't collide
with an RPC response by luck. **Cook 3 removes that safety:** a real NFC tap can fire at the
EXACT instant an RPC response is being written. Two threads writing the wire = interleaved
bytes = host reassembly explodes. This is the #1 Arena-forecast Cook 3 bug.

REQUIRED in Cook 3:
- A single-writer TX mutex on the FAP wire, held from the FIRST to the LAST fragment of ANY
  message (response OR broadcast). Do NOT release between fragments.
- OR keep the existing "worker never writes, only main thread sends" rule AND ensure the
  main thread serializes its own response-send vs broadcast-drain (they're both on the main
  thread, so a single send function with no preemption point between fragments suffices).
- Verify: the main-thread drain (cfc_drain_worker_results) and the RPC response path must
  not interleave. If both run on the main thread cooperatively, confirm neither yields
  mid-message.

This is the thing that passes the mock gate and corrupts the wire on the first real tap that
coincides with a response. Test it: subscribe, then hammer storage_read (multi-fragment
responses) WHILE tapping cards repeatedly. If reassembly errors appear, the interleave is real.
